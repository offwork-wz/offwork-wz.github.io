##### 공동구매 플랫폼 개발 (개인, 2026.04 ~ 진행 중)

**설명:** 높은 동시성이 발생하는 공동구매 환경에서 서비스 간 최종적 일관성 유지와 재고 정합성 확보를 목표로 개발 중인 MSA 기반 공동구매 플랫폼 (개인 프로젝트)

**개발 환경:** Java 21, Kotlin 2.1, Spring Boot 4.0.6, JPA (Hibernate), PostgreSQL 16, Kafka, Flyway, Docker Compose

* 마이크로서비스 아키텍처(MSA) 설계 및 데이터 격리  
  `product-service`, `user-service`, `groupbuy-service`(Java), `inventory-service`(Kotlin) 도메인 책임을 분리하고 Database-per-service 패턴을 적용하여 서비스별 데이터베이스를 독립적으로 운영
* Kafka 기반 비동기 이벤트 통신 및 Saga 패턴 적용  
  Kafka(KRaft Mode)를 기반으로 서비스 간 비동기 이벤트 통신 구조를 설계하고, 공동구매 신청 이벤트(`groupbuy.participation.requested`)를 시작으로 재고 차감 및 보상 트랜잭션을 수행하는 Saga 기반 워크플로우 구현
* 트랜잭션 커밋 이후 이벤트 발행 구조 설계  
  `@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)`를 활용하여 로컬 트랜잭션 커밋 이후에만 Kafka 이벤트를 발행하도록 구현함으로써 커밋 실패 시 발생 가능한 유령 이벤트 문제를 방지
* 비관적 락 기반 재고 동시성 제어  
  `@Lock(LockModeType.PESSIMISTIC_WRITE)`를 적용하여 선착순 공동구매 상황에서 발생할 수 있는 재고 경쟁 상태(Race Condition)를 제어하고 재고 수량의 정확성을 보장  
  짧은 트랜잭션 구간과 높은 재고 정합성 요구사항을 고려하여 비관적 락 방식을 채택
* Flyway 기반 데이터베이스 스키마 형상 관리  
  Flyway를 적용하여 마이크로서비스별 데이터베이스 스키마 변경 이력을 자동 관리하고 환경 간 일관된 배포 프로세스를 구축
